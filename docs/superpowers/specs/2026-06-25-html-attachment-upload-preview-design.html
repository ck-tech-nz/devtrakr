HTML 附件上传 + 文件卡片悬停预览

日期 2026-06-25 · 项目 DevTrack · 范围 描述 / 评论正文(Markdown 渲染路径) · 状态 待评审

1 · 背景与目标

用户向 Issue 描述区上传 HTML 文件时被拒绝,提示 不支持的文件类型: text/html。这类 HTML 多为"图文并茂的事实文档"(如根因梳理稿),既要能上传,也要能在正文里直接预览,而不必下载后才能看。

本设计交付两件事:

  1. 放行 HTML 上传 —— 把 text/html(及 .html/.htm 扩展名兜底)加入上传白名单。
  2. HTML 文件卡片 + 悬停预览 —— HTML 像其它文档一样渲染成文件卡片;鼠标悬停时,弹出沙箱 iframe 实时渲染完整 HTML。

已确认决策

① 预览形态:悬停弹出预览。 卡片外观与现有文件卡片一致(图标 + 文件名 + 徽章),悬停时弹层用沙箱 iframe 渲染完整 HTML —— 复用现有 .md 悬停预览的交互范式。

② iframe 隔离:sandbox="allow-scripts"(不带 allow-same-origin)。 脚本运行在 null 源,无法访问父页面、Cookie、本地存储,因此 JS 驱动的图表等仍能渲染,而风险被隔离。

③ 共享重构:抽出共享 composable。 现有 .md 悬停预览只存在于描述编辑器;评论区(MarkdownView.vue)完全没有。抽成共享逻辑后,评论区顺带获得 .md 与 .html 悬停预览。

3 · 后端改动 BACKEND

仅放行上传,不改对象存储的 MIME 映射(见安全说明)。

3.1 上传白名单

backend/apps/tools/services.py:

# ALLOWED_TYPES 增加
"text/html",
# EXTENSION_FALLBACK 增加(兜底个别浏览器上报空/异常 MIME)
"html", "htm",

安全:为什么不改 storage.py 的 EXT_TO_MIME

storage.py 已注释警告:绝不能让上传文件以 text/html 从公网 URL 提供 —— 否则任何人拿到公网链接即可触发存储型 XSS。因此:

  • HTML 在 MinIO 仍以 application/octet-stream + attachment 下发 → 直接访问公网链接只会下载,不会渲染。
  • 预览改走"前端 fetch 文本 → 沙箱 iframe srcdoc",完全不依赖公网 URL 的 content-type,从而既能预览又不暴露可渲染的 XSS 入口。
  • 数据库 mime_type 字段记录客户端上报的 text/html(仅作元数据,不影响下发);卡片分类按 URL 扩展名判定,与之解耦。

4 · 前端改动

4.1 白名单同步 FRONTEND

三处镜像的白名单同步加 text/html 与 html/htm:

  • MarkdownEditor.vue 的 ALLOWED_TYPES / EXTENSION_FALLBACK 及 <input accept>
  • pages/app/issues/[id].vue 的镜像常量(关联附件面板用)

4.2 HTML 文件卡片 FRONTEND

  • useMentionMarkdown.ts FILE_EXT_CATEGORY 增加 html:'html'、htm:'html'
  • 卡片样式新增 .md-file-html(🌐 图标 + 紫色 HTML 徽章),在 MarkdownEditor.vue 与 MarkdownView.vue 两处 CSS 同步(与现有六类保持一致)

4.3 悬停预览:抽共享逻辑 FRONTEND

把 MarkdownEditor.vue 现有的 .md 悬停预览(mdHover 状态、fetch+缓存、show/hide 定时、弹层定位、卡片绑定)抽出为可复用单元:

新单元职责依赖
useFileCardHoverPreview(rootRef, htmlGetter)
composable		对 .md 与 .html/.htm 卡片绑定 hover;管理 fetch+缓存+定时+定位;暴露响应式 hover 状态—
FileCardHoverPopup.vue
component		teleport 弹层。.md → markdown 渲染;.html/.htm → <iframe srcdoc sandbox="allow-scripts">上面的 hover 状态

MarkdownEditor.vue 与 MarkdownView.vue 都引用这套,移除 MarkdownEditor 里原有的内联 .md 悬停代码。

副带收益

评论区(MarkdownView.vue)此前对任何文件卡片都没有悬停预览;抽共享后,评论区的 .md 与 .html 卡片同样获得预览,消除了描述区与评论区的体验割裂。

4.4 iframe 渲染细节

  • 取内容:沿用现有 fetchMdContent 的跨源 fetch(MinIO 公网 URL,CORS 已放行),读 .text() 作为 srcdoc。
  • 隔离:sandbox="allow-scripts",不加 allow-same-origin → 脚本在 null 源,无法触达父页面 / Cookie / 存储。
  • 大小上限:渲染上限约 2MB。HTML 不能截断(截断会破坏标签结构),超限时弹层改提示"文件较大,请下载后查看"。
  • 点击行为:保持与其它文件卡片一致 —— 点击即下载(attachment);悬停才是预览入口。
