列表视图「显示/隐藏列」选择器

DevTrack · 问题跟踪 · 列表(表格)视图  |  设计稿  |  2026-06-29  |  状态:已确认,待实现

目标

在问题跟踪的列表视图工具栏增加一个「显示/隐藏表格列」按钮(弹出式),让用户自由控制表格列的显隐。ID、标题、状态 三列恒显(锁定),其余列可切换。当前显示的列集合即为默认值,用户的选择持久化到浏览器。

交互与外观

完全镜像现有的看板列编辑器 KanbanColumnEditor.vue(同一图标 i-heroicons-view-columns、同一 UPopover 模式),置于工具栏行尾。看板编辑器 v-if="viewMode==='kanban'",本组件 v-if="viewMode==='table'",二者互斥共用同一位置。

选择器内列锁定列分组在顶部(灰显、已勾选、不可点),其余按表格顺序排列。select(行选择复选框)列不参与配置,始终保留。

持久化

  • localStorage,沿用 issues:* 命名 → key issues:hidden-columns。
  • 存「被隐藏列 key 的数组」(JSON),与看板编辑器的 hidden 语义一致 —— 未来新增列默认可见,无需迁移已存偏好。
  • 默认值(未存或解析失败) = ['github_issues'],使开箱状态等于当前显示(GitHub Issues 默认隐藏,其余全显)。
  • 解析异常或 SSR 无 localStorage 时回退默认,沿用现有 loadPersistedFilters() 的 try/catch 写法。

父组件改动 app/pages/app/issues/index.vue

1) 列元数据(静态有序数组)

// 表格列定义:按既有表格顺序排列;locked 列恒显,其余可经列编辑器切换显隐
const TABLE_COLUMNS = [
  { key:'id', label:'ID', locked:true },
  { key:'title', label:'标题', locked:true },
  { key:'cause', label:'原因分析' },
  { key:'solution', label:'解决方案' },
  { key:'priority', label:'优先级' },
  { key:'status', label:'状态', locked:true },
  { key:'reporter', label:'提出人' },
  { key:'created_at', label:'历时' },
  { key:'estimated_completion', label:'要求完成日期' },
  { key:'github_issues', label:'GitHub Issues' },
]

单一数组,既有顺序:该数组即真实表格列顺序(标题→原因分析→解决方案→优先级→状态→…),columns computed 直接顺序遍历它生成表格,不改变现有视觉布局。状态 虽是锁定列但仍排在原位置。编辑器仅在展示时按 locked 标记把三个锁定列分组到顶部,不影响表格顺序。

2) columns computed 改为按隐藏集过滤

const columns = computed(() => {
  const cols = [{ id:'select', header:'', cell:'' }]
  for (const c of TABLE_COLUMNS) {              // 既有表格顺序
    if (!c.locked && hiddenColumns.value.includes(c.key)) continue
    cols.push({ accessorKey: c.key, header: c.label })
  }
  return cols
})

替换原本的静态列数组 + showGHColumn 条件。锁定列即使误入 hidden 数组也始终渲染。

3) 持久化的 ref + watch

const HIDDEN_COLUMNS_KEY = 'issues:hidden-columns'
function loadHiddenColumns(): string[] {
  if (typeof localStorage === 'undefined') return ['github_issues']
  try { const raw = localStorage.getItem(HIDDEN_COLUMNS_KEY)
    return raw ? JSON.parse(raw) : ['github_issues'] }
  catch { return ['github_issues'] }
}
const hiddenColumns = ref<string[]>(loadHiddenColumns())
watch(hiddenColumns, (v) => {
  if (typeof localStorage !== 'undefined')
    localStorage.setItem(HIDDEN_COLUMNS_KEY, JSON.stringify(v))
})

4) 工具栏挂载组件

<TableColumnEditor
  v-if="viewMode === 'table'"
  :columns="TABLE_COLUMNS"
  :hidden="hiddenColumns"
  @update:hidden="(v) => hiddenColumns = v"
/>

新组件 app/components/issue/TableColumnEditor.vue

  • Props:columns: {key,label,locked?}[]、hidden: string[]、size?。
  • Emit:update:hidden(string[])。
  • UPopover + UButton(i-heroicons-view-columns,variant="ghost",color="neutral",title/aria-label="显示/隐藏表格列")。
  • 渲染:锁定列分组(灰显、勾选、不可点、「必显」标签)→ 分隔线 → 可切换列(点击切换勾选)。
  • 切换逻辑:可见则把 key 加入 hidden 副本,隐藏则移除,emit('update:hidden', next)。

移除项

  • 表格上方的独立 GitHub Issues USwitch 块(index.vue 表头那一行)。
  • showGHColumn ref 及其在 columns 中的引用。

非目标 / 边界

  • 不做列拖拽排序(YAGNI),列顺序固定。
  • 不做「至少保留一列」守卫 —— 锁定列恒显,表格永不为空。
  • 隐藏列的 #xxx-cell / #xxx-header 插槽在该列不在 columns 时自动不渲染,无副作用;标题列拖拽列宽不受影响(标题恒显)。
  • 偏好存浏览器(localStorage),不同步到服务器/不跨设备 —— 符合需求「记录在浏览器」。

验证

  • npx nuxi typecheck 通过(无前端单测框架,组件为展示型,不做 TDD)。
  • 浏览器验证:列表视图出现按钮 → 勾选/取消即时增删列 → 刷新后保持 → 锁定列不可取消 → 切到看板视图按钮消失。
